Imputación de datos¶

La presencia de datos faltantes, es la situaci√≥n a la que permanentemente se enfrentan investigadores y tomadores de decisiones. Disponer de un archivo de datos completos es ideal, pero aplicar m√©todos de imputaci√≥n inapropiados para lograrlo, puede generar m√°s problemas de los que resuelve.

El prop√≥sito del presente notebook ser√° presentar un m√©todo de imputaci√≥n de datos faltantes haciendo uso de componentes principales. Para ello trabajaremos con una base de datos que contiene im√°genes de perros y gatos.

## funci√≥n de graficaci√≥n:
plotcd <- function(v){
  x <- matrix(v,64,64)
  image(1:65,1:65,t(apply(x,2,rev)),asp=1,xaxt="n",yaxt="n",
        col=grey((0:255)/255),ann=FALSE,bty="n")
}

## Visualizaci√≥n vector enmascarado
plotcdm <- function(v, m){
  ina <- which(m==0)
  w <- v
  w[ina] <- NA
  x <- matrix(w,64,64)
  image(1:65,1:65,t(apply(x,2,rev)),asp=1,xaxt="n",yaxt="n",
        col=grey((0:255)/255),ann=FALSE,bty="n")
}

## Funci√≥n de generaci√≥n de m√°scaras
## 0 en la m√°scara indica que el dato se omite
## 1 en la m√°scara indica que el dato se conserva
get.mask <- function(d1,# N√∫mero de filas
                     d2,# N√∫mero de columnas
                     p# probabilidad de omitir un dato
){
  return(matrix(sample(c(0,1),d1*d2,prob=c(p,1-p),replace=TRUE), d1, d2))
}

Aplicamos entonces el siguiente algoritmo para la reconstrucci√≥n de los datos:

In [9]:
n <- 90
matriz.proyec.n <- matriz.proyec[, 1:n]

### Reconstrucci√≥n
gatos.test.filled <- 0*gatos.test
for(igs in 1:nrow(gatos.test)){ # Para cada gato de prueba
  gato <- gatos.test[igs, , drop=FALSE]
  mask <- mask.gatos.test[igs, , drop=FALSE]
  i.mask <- (which(mask==1))
  g.mask <- gato[1, i.mask, drop=FALSE]
  u.mask <- matriz.proyec.n[i.mask, ]
  M <- t(u.mask)%*%u.mask
  f <- t(g.mask %*% u.mask)
  a <- solve(M,f)
  
  gatos.test.filled[igs,] <- t(matriz.proyec.n %*% a)
}

Imputación con múltiple iteraciones¶

A continuaci√≥n vamos a definir un algoritmo que realiza varias iteraciones que permiten mejorar la calidad de la imputaci√≥n. En cada iteraci√≥n, se calculan las componentes principales y se imputan los valores perdidos en los datos.

In [15]:
## Gatos enmarcarados al 10%
mask.gatos.train <- get.mask(nrow(gatos.train),
                            ncol(gatos.train),
                            p=0.1)

## Seleccionamos un gato de entrenamiento
i.gtrain <- sample(1:nrow(gatos.train),1)

En este ejemplo vamos a tomar por defecto 10 componentes principales para realizar el proceso de imputaci√≥n. Se define adem√°s un vector de errores, el cual es el resultado de contrastar el gato original contra cada gato resultante de cada iteraci√≥n. N√≥tese que en cada iteraci√≥n el resultado parece mejorar hasta cierto nivel.

for(iter in 1:20){

  for(igs in 1:nrow(gatos.train)){
    gato <- X[igs, , drop=FALSE]
    mask <- mask.gatos.train[igs, , drop=FALSE]
    i.mask <- (which(mask==1))
    g.mask <- gato[1, i.mask, drop=FALSE]
    u.mask <- matriz.proyec.n[i.mask, ]
    M <- t(u.mask)%*%u.mask
    f <- t(g.mask %*% u.mask)
    a <- solve(M,f)
    X_i[igs, ] <- t(matriz.proyec.n %*% a)
  }
  X[i.gap] <- X_i[i.gap]
